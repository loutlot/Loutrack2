# rigid_body_design_exploration

## 1. 前提と目的

- 既存: `waist` のみ実デザインあり（4 blob）
- 追加: 残り4剛体（`head`, `chest`, `left_foot`, `right_foot`）を設計
- 制約:
  - 各剛体は blob 4個
  - blob 直径は全て 14mm
  - 半球中心は `(0, 0, 7)`（mm）
  - 各 blob 中心は `||(x, y, z) - (0, 0, 7)|| <= 65` かつ `z > 7`
  - ただし `waist` は既存デザインを維持する

本資料では、プロジェクト要件（60fps目標、遮蔽耐性、誤認識抑制）に合わせて、
識別性とオクルージョン耐性を両立する5剛体組み合わせを提案する。

## 2. 剛体デザインの性能要件（本プロジェクト向け）

1. 幾何拘束成立:
   3点可視でも姿勢推定しやすいように、4点は非対称・非共面にする（1点は高z）。
2. 近接オクルージョン抑制:
   同一剛体内の最小点間距離は 55mm 以上を推奨（14mm球の重なり回避）。
3. 剛体識別性:
   6本の辺長シグネチャ（4点の組合せ距離）が剛体間で十分に離れること。
4. アルゴリズム整合:
   `Kabsch` 前段の対応付けで誤割当しにくい、鏡像対称を避けた形状にする。

### 3.2 制約チェック（設計値）

- `head/chest/left_foot/right_foot` は `max(||p - (0, 0, 7)||) <= 65mm` かつ `z > 7`
- `waist` は既存デザイン優先（上記制約の厳密適用対象外）
- 同一剛体内の最小点間距離: 約59mm〜73mm（14mm球に対して十分離間）

## 4. 識別性（距離シグネチャ）

4点から得る6本の辺長（昇順）を剛体IDの幾何ハッシュとして使用する。

- `waist`: [72.7, 75.9, 83.7, 104.0, 109.6, 118.4]
- `head`: [60.0, 60.3, 60.5, 61.0, 70.1, 74.4]
- `chest`: [59.1, 60.7, 68.0, 73.8, 82.7, 88.8]
- `left_foot`: [63.3, 67.4, 77.2, 79.7, 85.2, 101.4]
- `right_foot`: [60.3, 64.2, 65.5, 68.4, 74.5, 96.7]

設計時のポイント:
- `waist` は既存形状のままでも最も大きいスケールで他と強く分離
- `left_foot` と `right_foot` は鏡像にせず、固有シグネチャを分離
- `head` は比較的コンパクト、`chest` は中間スケールで差別化

## 5. アルゴリズム最適化（デザインを活かす）

1. データ対応付けの段階ゲート:
   `EKF予測 -> 2D Mahalanobis -> Epipolar -> Triangulation/Reproj -> 3D Mahalanobis -> Hungarian`
2. 剛体IDスコア:
   `score = w1*reproj + w2*kabsch_rms + w3*signature_error + w4*motion_consistency`
3. オクルージョン運用:
   - 3点可視: Kabsch継続
   - 2点以下: 予測姿勢(EKF)で保持、復帰時にヒステリシス付き再割当
4. DBSCANパラメータ再設計:
   現行 `eps=0.8*14mm` は剛体間分離用途として小さすぎるため、
   追跡予測ベースのゲーティング主体に寄せるか、`eps` を体サイズ帯へ再調整する。

## 6. 推奨する5剛体組み合わせ

- `head`: 提案座標
- `chest`: 提案座標
- `waist`: 既存座標を維持
- `left_foot`: 提案座標
- `right_foot`: 提案座標

この組み合わせは、半球制約・オクルージョン耐性・識別性のバランスが最も良い。
