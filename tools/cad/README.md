# CAD Tools

Generate first-pass printable marker fixtures from rigid-body marker coordinates.

The main entry point is `tools/cad/rigid_body_fixture.py`. It uses CadQuery to
turn marker-center coordinates into STL files for FDM print trials.

The bundled presets are exploratory coordinates from
`docs/10_in_progress/rigid_body_design.md`; they are useful for validating the
CAD pipeline but are not final manufacturing geometry.

## Setup

```sh
.venv/bin/python -m pip install -r tools/cad/requirements.txt
```

## Default Geometry

The default geometry is:

- 30 mm diameter, 4 mm high central cylinder at the origin.
- 1 mm chamfer on the top edge of the central cylinder.
- A paired hexagonal through-hole 4 mm from center and 5.2 mm round through-hole 10 mm in the opposite direction.
- The hole pair is automatically rotated toward the clearest direction away from stems.
- 14 mm diameter marker spheres at each coordinate.
- Marker centers are lifted 7 mm in Z during generation so marker spheres do not pass below the base bottom.
- 5 mm diameter stems from the closest point on a 25 mm diameter hub circle to each marker center, extended 1 mm into the hub side.
- 5 mm diameter balls on the extended hub-side stem endpoints.
- 1 mm blob-side stem fillets for smoother marker joints.

## Coordinate Handling

Input marker coordinates are interpreted as millimeters.

`--marker-z-lift` is applied only during CAD generation. It does not change the
tracking coordinates stored in docs or calibration data.

The current default `--marker-z-lift 7.0` exists because the marker spheres are
14 mm diameter. Lifting their centers by 7 mm prevents the lower half of a
sphere from going below the base-bottom plane when the source coordinate has a
low Z value.

## Hole Placement

The base has two through-holes:

- Hex hole: center offset 4 mm from origin, circumscribed around a 2.7 mm circle.
- Round hole: center offset 10 mm from origin in the opposite direction, diameter 5.2 mm.

The script chooses the shared hole axis automatically. It samples 0-360 degrees
and scores each candidate by clearance from:

- the base outer radius,
- every stem's XY-projected centerline,
- each stem radius,
- the round-hole radius,
- the hex-hole outer radius.

This is intended to keep the larger round hole from overlapping stems in bodies
such as `left_foot`.

## Generate a Sample STL

```sh
.venv/bin/python -m tools.cad.rigid_body_fixture --preset waist --out output/cad/waist_fixture.stl
```

To also export a CAD-editable STEP file:

```sh
.venv/bin/python -m tools.cad.rigid_body_fixture --preset waist --out output/cad/waist_fixture.stl --step output/cad/waist_fixture.step
```

## Generate All Seed Rigid Bodies

```sh
.venv/bin/python -m tools.cad.rigid_body_fixture --all-presets --out output/cad
```

This writes:

- `output/cad/head_fixture.stl`
- `output/cad/waist_fixture.stl`
- `output/cad/chest_fixture.stl`
- `output/cad/left_foot_fixture.stl`
- `output/cad/right_foot_fixture.stl`

## Custom Input JSON

You can generate from a JSON array:

```json
[
  [18.0, -50.0, 37.5],
  [0.6, 61.8, 20.2],
  [-6.6, -13.0, 63.3]
]
```

Run:

```sh
.venv/bin/python -m tools.cad.rigid_body_fixture --input path/to/body.json --out output/cad/custom_fixture.stl
```

For JSON with multiple bodies, pass `--body`:

```json
{
  "waist": {
    "marker_positions": [
      [18.0, -50.0, 37.5],
      [0.6, 61.8, 20.2],
      [-6.6, -13.0, 63.3]
    ]
  }
}
```

```sh
.venv/bin/python -m tools.cad.rigid_body_fixture --input path/to/bodies.json --body waist --out output/cad/waist_fixture.stl
```

## Useful Options

- `--marker-diameter`: marker sphere diameter in mm. Default: `14.0`.
- `--marker-z-lift`: temporary CAD-only Z lift for marker centers. Default: `7.0`.
- `--stem-diameter`: support stem diameter in mm. Default: `5.0`.
- `--hub-diameter`: circle diameter used for stem start points. Default: `25.0`.
- `--stem-base-extension`: how far each stem extends past the hub-side start point. Default: `1.0`.
- `--base-diameter`: central disk diameter in mm. Default: `30.0`.
- `--base-thickness`: central disk height in mm. Default: `4.0`.
- `--base-top-chamfer`: top-edge C chamfer on the disk. Default: `1.0`.
- `--hex-hole-inscribed-diameter`: circle diameter used to size the hex hole. Default: `2.7`.
- `--round-hole-diameter`: round through-hole diameter. Default: `5.2`.
- `--fillet-radius`: blob-side stem fillet radius. Default: `1.0`.

## Notes

- STL is the normal output for print trials.
- STEP output is optional via `--step`; it is useful for CAD inspection but is
  not generated by the all-presets command unless added later.
- The script is a prototype fixture generator. Confirm clearances, print
  orientation, strength, and marker mounting method before using a generated
  model as final hardware.
